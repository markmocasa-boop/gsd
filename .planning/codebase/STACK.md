# Technology Stack

**Analysis Date:** 2026-01-23

## Languages

**Primary:**
- JavaScript (ES2020+) - Installer, build scripts, and hooks
- Markdown - Prompts, commands, workflows, and reference documentation
- YAML - Command frontmatter and configuration

**Supporting:**
- JSON - Configuration files and data interchange

## Runtime

**Environment:**
- Node.js 16.7.0+ (specified in package.json engines)
- Available on Mac, Windows, and Linux

**Package Manager:**
- npm 8.0+ (implied by Node 16.7+)
- Lockfile: `package-lock.json` present

## Frameworks

**Core:**
- No framework dependencies - pure Node.js built-ins only

**Build/Dev:**
- esbuild 0.24.0 - Used to bundle and prepare hooks for distribution

**Build Process:**
- `npm run build:hooks` - Copies hook files to `hooks/dist/` before publishing
- No transpilation; source files are deployed as-is

## Key Dependencies

**Production:**
- None declared - zero production dependencies

**Development Only:**
- esbuild 0.24.0 - Build tool for bundling hooks

## Configuration

**Environment:**
- Installation locations determined by runtime:
  - Claude Code: `~/.claude/` (or `CLAUDE_CONFIG_DIR` env var)
  - OpenCode: `~/.config/opencode/` (or `OPENCODE_CONFIG_DIR`/`XDG_CONFIG_HOME`)
  - Local installs: `./.claude/` or `./.opencode/` (in project)

**Build:**
- No build configuration files (esbuild, tsconfig, webpack, etc.)
- Simple Node.js copy operations with path replacement

**Installation:**
- `bin/install.js` - Entry point for `npx get-shit-done-cc` command
- Handles file copying, directory creation, settings.json configuration
- Supports both Claude Code and OpenCode runtimes

## Platform Requirements

**Development:**
- Node.js 16.7.0 or higher
- `git` (for cloning the repo)

**Runtime (User Machines):**
- Node.js 16.7.0+ (same requirement)
- One of: Claude Code or OpenCode IDE installed
- File system access to home directory and project directory
- Network access to npm registry (for version checking via `gsd-check-update.js`)

## Package Entry Point

**CLI:**
- `bin/install.js` - Executable script registered as `get-shit-done-cc` bin command
- Installed globally via `npx get-shit-done-cc` or locally

## Distribution

**Published as:**
- npm package: `get-shit-done-cc` at https://www.npmjs.com/package/get-shit-done-cc
- Files included in npm publish:
  - `bin/` - Installation script
  - `commands/` - GSD slash commands for Claude Code
  - `get-shit-done/` - Workflows, templates, and reference docs
  - `agents/` - Subagent prompt files
  - `hooks/dist/` - Compiled hooks (generated by build)
  - `scripts/` - Build scripts
  - `CHANGELOG.md` - Version history
